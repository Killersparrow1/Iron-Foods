# Governance

Iron Foods is a community-governed open-source project.

## Decision Making
- Decisions are made through consensus among core maintainers.
- For major architectural changes or dataset policy shifts, an RFC (Request for Comments) process will be used.
- Dataset additions are reviewed by regional experts (where applicable) and core maintainers.

## Maintainers
- Maintainers are selected based on sustained contributions to the project, demonstrating a commitment to data quality and community health.
- To become a maintainer, you must first be an active contributor for at least 3 months.

## Disputes
- Disputes that cannot be resolved via consensus will be decided by a majority vote of the core maintainer team.
- The project founder retains tie-breaking authority.

## Dataset & Schema Changes
- Schema changes require approval from at least two core maintainers.
- Dataset additions are welcome via Pull Requests and are subject to the [Review Process](REVIEW_PROCESS.md).
