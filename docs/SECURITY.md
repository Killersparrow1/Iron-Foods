# Security

## Reporting Vulnerabilities
If you discover a security vulnerability in Iron Foods, please report it privately via email to the maintainers rather than opening a public issue.

## Security Practices
- All data submissions are validated against a strict schema.
- We do not store any user-sensitive information.
- All contributions are reviewed by maintainers to prevent malicious data injection.

# Source Policy

We prioritize transparency and traceability for all nutrition data.

## Source Tiering
- **Tier 1 (Institutional):** IFCT (Indian Food Composition Tables)
- **Tier 2 (Government/International):** USDA FoodData Central
- **Tier 3 (Academic):** Peer-reviewed research papers
- **Tier 4 (Community):** Verified community-submitted data (requires maintainer review)

We require traceable source references for all submissions.
